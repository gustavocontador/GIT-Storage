## 1. VISAO GERAL DA JORNADA

### O que e o BranDesire?

Um framework de **16 campos** que captura a essencia de uma marca de forma profunda, usando o **metodo unniko** (Feynman + 5 Whys) para garantir que as respostas nao sejam superficiais.

### Fluxo Macro (10 Stages)

```
USUARIO ENTRA
      |
[STAGE 1] Atendimento .......... 16 campos de input (briefing)
      |
[STAGE 2] Search & Insights .... pesquisa de mercado automatica
      |
[STAGE 3] DEENEA ............... essencia (proposito, arquetipos, posicionamento)
      |
[GATE 1]  CBO Gate ............. validacao estrategica (12 itens, 100%)
      |
[STAGE 5] UNNIKO ............... identidade (visual, verbal, sensorial, design system)
      |
[GATE 2]  CBO Gate ............. validacao de identidade (12 itens, 100%)
      |
[STAGE 7] storYOURtelling ...... narrativa (big idea, campanha, tom de voz)
      |
[GATE 3]  CBO Gate ............. validacao criativa (13 itens, 100%)
      |
[STAGE 9] Operations ........... content-ops + growth-ops (paralelo)
      |
[GATE 4]  CBO Gate ............. entrega final (16 itens, 100%)
      |
MARCA CONSTRUIDA
```

**Tempo estimado do processo completo:** 25-45 dias uteis

## 4. ANALISE DE FRICCOES

### 4.1 Friccoes de ENTRADA (Stage 1 — Briefing)

| # | Friccao | Impacto | Severidade |
|---|---------|---------|------------|
| F1 | **79 perguntas potenciais e muita coisa** — pode intimidar o usuario e gerar abandono | Desistencia no inicio do processo | ALTA |
| F2 | **Todos os campos sao texto livre** — sem guias, sem exemplos, sem opcoes pre-definidas | Respostas vagas ou "travadas" | ALTA |
| F3 | **Nao ha salvar progresso** — se o usuario parar no meio, perde tudo | Frustracao e retrabalho | ALTA |
| F4 | **Nao ha indicador de progresso** — usuario nao sabe onde esta nem quanto falta | Ansiedade, sensacao de processo infinito | MEDIA |
| F5 | **Metodo unniko pode parecer interrogatorio** — 5 Whys repetidos podem irritar | Experiencia negativa | MEDIA |
| F6 | **Blocos Execucao (13-14) podem assustar** — timeline e budget logo no comeco | Usuario pode nao ter essas respostas ainda | MEDIA |
| F7 | **Sem onboarding** — usuario comeca direto nas perguntas sem entender o processo | Falta de contexto | MEDIA |

### 4.2 Friccoes de PROCESSO (Stages 2-9)

| # | Friccao | Impacto | Severidade |
|---|---------|---------|------------|
| F8 | **Caixa preta entre stages** — usuario nao ve o que acontece entre briefing e entregaveis | Ansiedade, falta de confianca | ALTA |
| F9 | **4 CBO Gates com 100% obrigatorio** — um unico item reprovado trava todo o processo | Gargalo potencial, atraso significativo | ALTA |
| F10 | **Nao ha preview parcial** — usuario so ve resultado quando stage termina | Expectativa desalinhada | MEDIA |
| F11 | **Sem feedback loop intermediario** — usuario nao pode ajustar rumo durante processamento | Risco de retrabalho no final | MEDIA |
| F12 | **25-45 dias para resultado completo** — muito tempo para quem tem urgencia | Perda de clientes com urgencia | MEDIA |
| F13 | **Entregaveis sao documentos markdown** — formato tecnico, pouco visual para cliente | Percepcao de valor baixa | MEDIA |

### 4.3 Friccoes de SAIDA (Entregaveis)

| # | Friccao | Impacto | Severidade |
|---|---------|---------|------------|
| F14 | **Excesso de documentos** — dezenas de entregaveis podem confundir o cliente | Dificuldade de usar na pratica | MEDIA |
| F15 | **Sem dashboard unificado** — cada entregavel e um arquivo separado | Visao fragmentada | MEDIA |
| F16 | **Sem guia de implementacao** — cliente recebe a marca mas nao sabe como usar | Marca fica na gaveta | ALTA |

## 5. OPORTUNIDADES DE MELHORIA

### 5.1 Quick Wins (implementacao rapida, alto impacto)

| # | Oportunidade | Resolve Friccao | Esforco | Impacto |

| O5 | **Reordenar blocos** — mover Execucao (13-14) para o final ou tornar opcional | F6 | Baixo | MEDIO |

## 7. RECOMENDACOES PARA TRANSFORMACAO EM APP

### 7.1 Principios de Design para o App

1. **Progressivo** — nunca mostrar tudo de uma vez. Um bloco por vez, com contexto.
2. **Visual** — substituir texto por opcoes visuais sempre que possivel (selecionar arquetipo por imagem, nao por texto).
3. **Transparente** — o usuario sempre sabe onde esta, o que falta, quanto tempo.
4. **Flexivel** — permitir pular e voltar. Nem todo cliente tem todas as respostas agora.
5. **Recompensador** — cada etapa concluida mostra um resultado parcial. Nunca "caixa preta".

### 7.2 Estrutura Sugerida do App

```
[1. ONBOARDING]
    "O que e construir uma marca forte?"
    2 min — video ou carrossel interativo
    Resultado: usuario entende o valor do processo
         |
[2. BRIEFING INTELIGENTE]
    Modo conversacional (chatbot-style)
    16 campos em 8 blocos
    Cada bloco = 1 tela
    Opcoes pre-definidas + texto livre
    Exemplos contextuais
    Salvar rascunho automatico
    Indicador: "Bloco 3/8 — Publico"
         |
[3. DIAGNOSTICO INSTANTANEO]
    Apos briefing, mostra Brand Score
    Grafico radar: Essencia / Identidade / Narrativa
    "Sua marca esta 45% construida"
    Recomendacoes personalizadas
         |
[4. JORNADA GUIADA]
    Timeline visual com stages
    Cada stage mostra:
      - O que vai acontecer
      - Quanto tempo leva
      - Preview do entregavel
    Updates em tempo real
         |
[5. ENTREGAVEIS VISUAIS]
    Dashboard com tabs:
      - Estrategia (DEENEA)
      - Identidade (UNNIKO)
      - Narrativa (storYOURtelling)
      - Conteudo (Content-Ops)
      - Metricas (Growth-Ops)
    Cada entregavel: visual + download PDF
         |
[6. GUIA DE USO]
    "Como usar sua marca"
    Checklists por canal (Instagram, site, etc.)
    Exemplos de aplicacao
    Brand Book interativo
```

### 7.3 Modos de Uso do App

| Modo | Para quem | O que inclui | Duracao |
|------|-----------|-------------|---------|
| **Express** | Urgencia / marca simples | Briefing + DEENEA + storYOURtelling (pula UNNIKO detalhado) | 5-10 dias |
| **Completo** | Marca que quer tudo | Jornada completa (10 stages) | 25-45 dias |
| **Modular** | Marca que ja tem algo | Escolher stages especificos (ex: so UNNIKO) | Variavel |
| **DIY** | Empreendedor solo | Briefing guiado + templates pre-preenchidos | 1-3 dias |

## RESUMO EXECUTIVO

### Numeros-chave

- **16 campos** de input do usuario
- **79 perguntas** potenciais no briefing
- **10 stages** na jornada completa
- **4 checkpoints** tecnicos + **4 CBO gates** executivos
- **42 agentes** envolvidos
- **20+ frameworks** academicos
- **25-45 dias** de processo completo
- **16 friccoes** identificadas (5 de severidade ALTA)
- **15 oportunidades** de melhoria mapeadas

### Top 5 Acoes Prioritarias

1. **Modo conversacional progressivo** no briefing (resolve 3 friccoes de uma vez)
2. **Salvar rascunho automatico** (previne perda de trabalho)
3. **Timeline visual** com updates em tempo real (elimina a "caixa preta")
4. **Previews incrementais** a cada stage (mantem engajamento)
5. **Diagnostico instantaneo** pos-briefing (recompensa imediata)
