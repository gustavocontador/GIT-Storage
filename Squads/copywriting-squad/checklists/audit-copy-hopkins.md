# Checklist de Audit de Copy - Claude Hopkins

## Instrucoes

Use este checklist para auditar copy existente antes de publicar ou quando resultados estao abaixo do esperado.

Marque cada item como:
- [x] OK - Atende ao criterio
- [ ] AJUSTAR - Precisa de melhoria
- [N/A] - Nao aplicavel

---

## 1. HEADLINE

### Promessa
- [ ] Promete beneficio especifico (nao generico)?
- [ ] O beneficio e relevante para o publico-alvo?
- [ ] A promessa e crivel (nao exagerada)?

### Selecao
- [ ] Seleciona o publico certo?
- [ ] Repele quem nao e cliente ideal?
- [ ] Fala diretamente com a dor/desejo?

### Curiosidade
- [ ] Cria vontade de ler mais?
- [ ] Tem elemento de novidade?
- [ ] Evita cliches gastos?

### Testabilidade
- [ ] Seria facil criar variante para teste A/B?
- [ ] Tem elemento mensuravel?

**SCORE HEADLINE:** ___/10

---

## 2. LEAD (ABERTURA)

### Continuidade
- [ ] Mantem a promessa da headline?
- [ ] Transicao e suave (nao abrupta)?
- [ ] Expande a curiosidade?

### Conexao
- [ ] Conecta emocionalmente nas primeiras linhas?
- [ ] Mostra que entende o leitor?
- [ ] Cria identificacao?

### Fluxo
- [ ] Leva naturalmente para proxima secao?
- [ ] Sem pontos de saida prematuros?
- [ ] Mantem momentum?

**SCORE LEAD:** ___/10

---

## 3. BODY COPY

### Especificidade
- [ ] Usa numeros especificos vs generalidades?
- [ ] Detalhes concretos vs vagos?
- [ ] Exemplos especificos vs abstratos?

**Exemplos de problema:**
- FRACO: "Muitos resultados" -> FORTE: "347 casos documentados"
- FRACO: "Economize tempo" -> FORTE: "Economize 4.7 horas por semana"

### Provas
- [ ] Cada claim importante tem prova?
- [ ] Provas sao verificaveis?
- [ ] Mix de tipos de prova (dados, testimoniais, estudos)?

### Fluxo Logico
- [ ] Argumentos seguem ordem logica?
- [ ] Cada paragrafo leva ao proximo?
- [ ] Sem saltos confusos de assunto?

### Objecoes
- [ ] Principais objecoes sao endere√ßadas?
- [ ] Objecoes sao antecipadas (antes do leitor pensar)?
- [ ] Respostas sao convincentes?

### Legibilidade
- [ ] Paragrafos curtos (2-4 linhas)?
- [ ] Frases curtas e diretas?
- [ ] Vocabulario acessivel (nivel 5a serie)?
- [ ] Subtitulos quebram texto longo?
- [ ] Bullets facilitam escaneamento?

**SCORE BODY:** ___/10

---

## 4. OFERTA

### Clareza
- [ ] Oferta e clara e simples de entender?
- [ ] Esta obvio o que a pessoa recebe?
- [ ] Preco e apresentacao sao claros?

### Valor Percebido
- [ ] Valor percebido e maior que preco?
- [ ] Stack de valor esta bem apresentado?
- [ ] Bonus agregam valor real?

### Urgencia
- [ ] Urgencia e real (nao artificial)?
- [ ] Razao para urgencia esta explicada?
- [ ] Deadline e especifico?

### Garantia
- [ ] Garantia remove risco do comprador?
- [ ] Garantia e especifica (nao generica)?
- [ ] Termos estao claros?

**SCORE OFERTA:** ___/10

---

## 5. CALL TO ACTION (CTA)

### Unicidade
- [ ] Ha UMA acao clara principal?
- [ ] Nao ha CTAs competindo?
- [ ] Acao e inequivoca?

### Facilidade
- [ ] Facil de executar a acao?
- [ ] Passos estao claros?
- [ ] Friccao minimizada?

### Repeticao
- [ ] CTA aparece em pontos estrategicos?
- [ ] Nao exagera (muito CTA irrita)?
- [ ] CTA final e forte?

**SCORE CTA:** ___/10

---

## 6. VALIDACAO GERAL

### Teste do Vendedor
- [ ] Se um vendedor falasse isso pessoalmente, funcionaria?
- [ ] A copy soa natural quando lida em voz alta?
- [ ] Nao e "marketese" robotico?

### Testabilidade
- [ ] Cada elemento principal e testavel separadamente?
- [ ] Ha tracking configurado?
- [ ] Pode medir resultados claramente?

### Alinhamento de Awareness
- [ ] Copy combina com nivel de consciencia do publico?
- [ ] Extensao esta apropriada para awareness?
- [ ] Tom esta alinhado?

**SCORE GERAL:** ___/10

### Interpretacao

- **50-60:** Copy excelente, pronta para testar
- **40-49:** Boa, mas precisa ajustes pontuais
- **30-39:** Precisa revisao significativa
- **< 30:** Reescrever com mais estrategia

---

## PROXIMOS PASSOS

### Se score < 40:
1. Identificar os 3 piores scores
2. Reescrever essas secoes usando clone apropriado
3. Re-auditar antes de publicar

### Se score >= 40:
1. Ajustar pontos marcados como "AJUSTAR"
2. Aplicar 5-10 triggers Sugarman relevantes
3. Publicar e comecar testes A/B

### Testes Prioritarios (em ordem):
1. Headline (maior impacto)
2. Oferta (segundo maior impacto)
3. Lead (terceiro maior impacto)
4. CTA (quarto maior impacto)
