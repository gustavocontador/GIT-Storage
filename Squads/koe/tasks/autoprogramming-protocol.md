## Propósito

> "You don't change habits. You change identity. And identity changes
> when evidence accumulates. Autoprogramming is the deliberate practice
> of stacking evidence until the new identity becomes the default."
> — Dan Koe

Executar o protocolo de Autoprogramming — reprogramação deliberada de identidade
através de acumulação estratégica de evidência. Diferente de afirmações vazias,
este protocolo constrói a nova identidade através de PROVA, não de palavras.

## Contexto

Autoprogramming é o termo de Dan Koe para o processo de reprogramação de identidade
consciente. A premissa é simples mas profunda:

### O Modelo de Mudança de Identidade
```
Crença atual → Ação deliberada → Evidência → Nova crença → Nova identidade
     ↑                                                           |
     └───────── (Identidade reforça crenças, loop contínuo) ─────┘
```

### Por que Afirmações NÃO Funcionam (e Evidência SIM)
- **Afirmação**: "Eu sou disciplinado" (sem prova = dissonância cognitiva)
- **Evidência**: "Eu acordei às 5h por 30 dias seguidos" (prova = crença natural)

O cérebro não acredita no que você DIZ — acredita no que você FAZ repetidamente.

### Os 3 Tipos de Evidência
1. **Evidência de Ação**: "Eu FIZ isso" (comportamento)
2. **Evidência de Resultado**: "Eu CONSEGUI isso" (outcome)
3. **Evidência de Reconhecimento**: "Outros VEEM isso em mim" (validação externa)

### Duração para Mudança de Identidade
- **21 dias**: O comportamento se torna familiar
- **66 dias**: O comportamento se torna automático
- **90 dias**: Outros começam a notar a mudança
- **1 ano**: A nova identidade é indistinguível de "quem você sempre foi"

## Steps

### Step 1 — Identificar Target Identity (10min)

Defina com precisão quem você quer se tornar:

**Identidade Atual** (quem você é hoje):
Responda honestamente:
1. Como você se descreve quando está sozinho? ____
2. Que rótulos você carrega? (positivos e negativos) ____
3. O que você acredita ser verdade sobre suas capacidades? ____
4. Que frase começa com "eu sou..." que te limita? ____
5. Que frase começa com "eu nunca..." que te define? ____

**Target Identity** (quem você quer ser):
1. Que tipo de pessoa realiza o que você deseja? ____
2. Que adjetivos descrevem essa pessoa? ____
3. O que essa pessoa acredita sobre si mesma? ____
4. Que hábitos essa pessoa tem? ____
5. Como essa pessoa reage sob pressão? ____

**Identity Gap Statement**:
> "Eu ATUALMENTE me vejo como [ATUAL], mas quero ser [TARGET]."
> "A principal diferença entre as duas identidades é [GAP]."

**Validação da target identity**:
- [ ] É aspiracional mas alcançável?
- [ ] Você conhece alguém que JÁ É isso?
- [ ] A mudança está sob seu controle (não depende de outros)?
- [ ] Você genuinamente quer isso (não é pressão externa)?

### Step 2 — Auditar Identity Gaps (10min)

Mapeie as lacunas específicas entre identidade atual e target:

**Gap Analysis**:

| Dimensão | Identidade Atual | Target Identity | Gap | Prioridade |
|----------|-----------------|-----------------|-----|-----------|
| Crenças | | | | 1-5 |
| Hábitos | | | | 1-5 |
| Skills | | | | 1-5 |
| Relações | | | | 1-5 |
| Ambiente | | | | 1-5 |
| Linguagem | | | | 1-5 |
| Aparência | | | | 1-5 |
| Rotina | | | | 1-5 |

**Top 3 Gaps por Prioridade**:
1. Gap: ____ | Porque é prioritário: ____
2. Gap: ____ | Porque é prioritário: ____
3. Gap: ____ | Porque é prioritário: ____

**Crenças Limitantes que Sustentam os Gaps**:
Para cada gap, identifique a crença que o mantém:
1. Gap 1 → Crença: "____"
2. Gap 2 → Crença: "____"
3. Gap 3 → Crença: "____"

### Step 3 — Criar Identity Evidence File (10min)

O Identity Evidence File é o documento onde você acumula PROVAS da nova identidade.

**Estrutura do Evidence File**:

```markdown
# Identity Evidence File
## Target Identity: [DESCRIÇÃO]
## Iniciado em: [DATA]
## Checkpoint: Dia 66

### Evidências de Ação (coisas que FIZ)
- [DATA] ____
- [DATA] ____

### Evidências de Resultado (coisas que CONSEGUI)
- [DATA] ____
- [DATA] ____

### Evidências de Reconhecimento (o que outros VIRAM)
- [DATA] ____
- [DATA] ____

### Contagem Total de Evidências: ____
```

**Primeiras 5 evidências para coletar esta semana**:
1. Ação: Eu posso ____ esta semana para provar que sou [TARGET]
2. Ação: Eu posso ____ esta semana para provar que sou [TARGET]
3. Ação: Eu posso ____ esta semana para provar que sou [TARGET]
4. Ação: Eu posso ____ esta semana para provar que sou [TARGET]
5. Ação: Eu posso ____ esta semana para provar que sou [TARGET]

**Regras do Evidence File**:
- Registrar DIARIAMENTE (mínimo 1 evidência por dia)
- Reler semanalmente (aos domingos, releia todas as evidências)
- Nunca deletar evidências (mesmo que pareçam pequenas)
- Cada evidência começa com verbo de ação ("Fiz", "Completei", "Criei")

### Step 4 — Design Daily Reinforcement Practice (10min)

Crie a prática diária de reforço da nova identidade:

**Ritual Matinal (5-10 min)**:
1. **Identity Statement** (30s): Ler em voz alta:
   > "Eu sou [TARGET IDENTITY]. A evidência disso é [ÚLTIMA EVIDÊNCIA]."

2. **Evidence Scan** (2min): Reler as 3 últimas evidências do arquivo

3. **Daily Intention** (1min): Definir 1 ação que gera evidência hoje:
   > "Hoje eu vou ____ como prova de que sou [TARGET]."

4. **Polarity Check** (1min): Lembrar da anti-visão se NÃO mudar

5. **Visualization** (1-2min): Visualizar-se agindo como [TARGET] na situação
   mais importante do dia

**Ritual Noturno (3 min)**:
1. **Evidence Logging** (1min): Registrar as evidências do dia
2. **Identity Confirmation** (1min): "Hoje eu fui [TARGET] quando ____"
3. **Tomorrow Prep** (1min): Preparar a ação de evidência de amanhã

**Weekly Review (15 min, Domingo)**:
1. Reler TODAS as evidências da semana
2. Contar total de evidências acumuladas
3. Avaliar: "Me sinto mais como [TARGET]?" (1-10)
4. Ajustar ações da próxima semana se necessário

### Step 5 — Definir Checkpoint de 66 Dias (5min)

O checkpoint de 66 dias é baseado na pesquisa de Phillippa Lally (UCL) sobre
formação de hábitos automáticos.

**Métricas para o Dia 66**:

1. **Evidence Count Target**: ≥ 66 evidências registradas
2. **Identity Score**: Autoavaliação ≥ 7/10 ("Me sinto como [TARGET]")
3. **External Validation**: ≥ 3 pessoas notaram a mudança
4. **Automaticity**: Comportamentos target acontecem sem esforço consciente
5. **Belief Shift**: As crenças limitantes do Step 2 enfraqueceram

**Checkpoint Questions (Dia 66)**:
- [ ] Acumulei ≥ 66 evidências?
- [ ] Me sinto naturalmente como [TARGET]?
- [ ] Outros percebem a mudança?
- [ ] Os comportamentos são automáticos?
- [ ] As crenças limitantes mudaram?

**Se SIM para ≥ 4**: A nova identidade está instalada. Manutenção mensal.
**Se SIM para 2-3**: Estender protocolo por mais 30 dias.
**Se SIM para ≤ 1**: Revisar target identity (pode não ser genuína).
