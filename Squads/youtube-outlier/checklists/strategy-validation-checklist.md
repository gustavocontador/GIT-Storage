# Strategy Validation Checklist

Use this checklist to validate any YouTube strategy before delivery. Every strategy must pass through these quality gates.

## Pre-Strategy Checks

- [ ] Channel audit completed (OO_QG_002 — BLOCKING)
- [ ] Target audience confirmed (The Working Professional)
- [ ] Channel stage identified (new/growing/established/scaling)
- [ ] Time budget confirmed (default: 4 hours/week)
- [ ] Faceless requirement confirmed

## Strategy Quality

### Actionability
- [ ] Every recommendation has a specific action (not vague advice)
- [ ] Actions are realistic for the time budget
- [ ] Actions are sequenced by priority (what to do first)
- [ ] No action requires showing face on camera
- [ ] Timeline is realistic (6-month minimum for results)

### Data Backing
- [ ] Recommendations are backed by data (not opinions)
- [ ] Benchmarks are niche-specific (not generic YouTube averages)
- [ ] Metrics are actionable (not vanity metrics)
- [ ] Comparison baseline is stated

### Policy Compliance (OO_QG_003 — BLOCKING)
- [ ] All strategies comply with YouTube Community Guidelines
- [ ] All strategies comply with Advertiser-Friendly Guidelines
- [ ] AI content disclosure requirements addressed
- [ ] No strategies that violate Reused Content policy
- [ ] Monetization requirements accounted for

### Voice DNA
- [ ] Strategy delivered in a direct, no-BS tone
- [ ] No guru language used
- [ ] No false promises or guaranteed results
- [ ] Honest about timelines and effort required
- [ ] Family-first values maintained (4 hrs/week, keep your job)

### Completeness
- [ ] Strategy has clear phases with milestones
- [ ] KPIs defined for each phase
- [ ] Weekly action items specified
- [ ] Risk factors acknowledged
- [ ] Review cadence established

## Post-Strategy Checks

- [ ] User understands next steps
- [ ] Handoff to relevant specialist is clear
- [ ] Analytics tracking plan in place
- [ ] First review date scheduled

## The Voice Test

Before delivering any strategy, ask: "Does this match the voice DNA?"

- Is this actionable? (Not just theory)
- Is this honest? (No hype or false promises)
- Is this safe? (Doesn't risk the user's job/family)
- Is this smart? (Bridge strategy, not cliff jumping)

If any answer is "no" — revise before delivery.
