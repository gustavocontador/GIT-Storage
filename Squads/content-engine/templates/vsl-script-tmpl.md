# VSL (Video Sales Letter) Script Template

**Pattern:** CP-PP-006
**Source:** Stefan Georgi (RMBC Method)

---

## Pre-Requisitos (RMBC)

Antes de escrever o VSL:

- [ ] Research completa (publico, dores, desejos, objecoes)
- [ ] Mechanism definido e nomeado
- [ ] Brief preenchido (todas secoes)

---

## VSL Structure

### PARTE 1: HOOK (0:00 - 0:30)

```
[Pattern interrupt - pare o scroll IMEDIATAMENTE]

Opcoes de hook:
A) Resultado chocante: "[Resultado] em [prazo curto]"
B) Pergunta provocadora: "Voce [situacao dolorosa]?"
C) Declaracao ousada: "[Crenca popular] esta ERRADO"
D) Historia instantanea: "Quando [momento dramatico]..."

REGRA: Se nao prende em 5 segundos, ja perdeu.
```

### PARTE 2: PROBLEMA (0:30 - 3:00)

```
[Amplificacao da dor em 3 camadas]

Camada 1 - Superficie:
"Voce [sintoma visivel]..."

Camada 2 - Emocional:
"E no fundo voce sabe que [medo real]..."

Camada 3 - Existencial:
"A verdade e que se nada mudar, daqui a [prazo] voce vai [consequencia]..."

[Mostre que voce ENTENDE]
"Eu sei como e porque [prova de empatia]..."
```

### PARTE 3: TRANSICAO (3:00 - 4:00)

```
[De dor para esperanca - mas com credibilidade]

"Mas e se existisse uma forma de [resultado] que fosse completamente diferente
de tudo que voce ja tentou?"

"Nao estou falando de [solucao generica A].
Nem de [solucao generica B].
Estou falando de algo que [diferencial]."
```

### PARTE 4: MECHANISM REVEAL (4:00 - 8:00)

```
[O coracao do VSL - por que FUNCIONA diferente]

Nome do mecanismo: [NOME PROPRIETARIO]

"Funciona assim:"

Passo 1: [Acao simples] ‚Üí [Resultado parcial]
Passo 2: [Acao simples] ‚Üí [Resultado parcial]
Passo 3: [Acao simples] ‚Üí [Resultado completo]

"E por isso que e diferente de [alternativas]:
- [Alternativa A] faz [X], nos fazemos [Y]
- [Alternativa B] depende de [Z], nos eliminamos isso"

[Visual: Se possivel, mostre diagrama/processo]
```

### PARTE 5: CREDIBILIDADE / PROVA (8:00 - 12:00)

```
[Prove que funciona - nao com claims, com evidencia]

Prova pessoal:
"Eu [historia de resultado com numeros]"

Prova social (3+ depoimentos):
"[Nome] estava [antes]. Depois de [prazo], [resultado com numero]."
"[Nome]: '[Quote direto do cliente]'"

Prova logica:
"Funciona porque [razao cientifica/logica]"

[Se possivel: screenshots, prints, dados]
```

### PARTE 6: OFFER (12:00 - 16:00)

```
[Apresente tudo que eles recebem]

"Aqui esta o que voce vai receber:"

‚úÖ [Item 1]: [Nome] - [beneficio] ........... Valor: R$ XXX
‚úÖ [Item 2]: [Nome] - [beneficio] ........... Valor: R$ XXX
‚úÖ [Item 3]: [Nome] - [beneficio] ........... Valor: R$ XXX

"Bonus especial:"
üéÅ [Bonus 1]: [Nome] - [beneficio] ......... Valor: R$ XXX
üéÅ [Bonus 2]: [Nome] - [beneficio] ......... Valor: R$ XXX

"Valor total: R$ X.XXX"
"Voce nao paga nem perto disso."
"Seu investimento hoje: R$ XXX"
```

### PARTE 7: GUARANTEE (16:00 - 17:00)

```
"E pra que voce nao tenha NENHUMA duvida..."

"Garantia [Nome da Garantia]:"
"Se em [prazo] voce nao [resultado minimo],
eu devolvo cada centavo. Sem perguntas."

"O risco e TODO meu. Zero risco pra voce."
```

### PARTE 8: CLOSE (17:00 - 20:00)

```
[Urgencia + CTA + Fantasma emocional]

"Voce tem duas opcoes agora:"

"Opcao 1: Fechar esse video e continuar [situacao atual].
Daqui a [prazo], voce vai estar exatamente no mesmo lugar.
Mesma frustracao. Mesmo resultado."

"Opcao 2: Clicar no botao abaixo e [resultado].
Em [prazo], voce vai [transformacao]."

"A escolha e sua. Mas o tempo nao para."

[CTA: Botao com verbo + resultado]
"Clique aqui para [resultado]"

[Escassez se aplicavel]
"So ate [deadline]. Depois, [consequencia]."
```

## Timing Guidelines

| Secao     | Duracao    | % do VSL |
| --------- | ---------- | -------- |
| Hook      | 30s        | 3%       |
| Problema  | 2:30       | 15%      |
| Transicao | 1:00       | 5%       |
| Mecanismo | 4:00       | 25%      |
| Prova     | 4:00       | 25%      |
| Oferta    | 4:00       | 20%      |
| Garantia  | 1:00       | 5%       |
| Close     | 3:00       | 15%      |
| **Total** | **~20min** | **100%** |

---

## Checklist Pre-Torriani

- [ ] Hook prende em 5 segundos?
- [ ] Dor em 3 camadas (superficie ‚Üí emocional ‚Üí existencial)?
- [ ] Mecanismo unico nomeado e explicavel visualmente?
- [ ] 3+ provas sociais com numeros?
- [ ] Offer stack com valor empilhado?
- [ ] Garantia clara e sem risco pro comprador?
- [ ] Close com dualidade (2 opcoes) e fantasma emocional?
- [ ] Zero jargao corporativo?
- [ ] Concorrente poderia usar o mesmo VSL? (se sim, refaz)
