# Sales Page Template

**Pattern:** CP-PP-002
**Source:** Gary Halbert (Boron Letters) + Stefan Georgi (RMBC)

---

## Structure

### 1. PRE-HEAD (Optional)

```
[Calling out the audience specifically]
"Para [publico especifico] que [situacao atual]..."
```

### 2. HEADLINE

```
[Promessa principal - beneficio #1 + mecanismo unico]
[Use numeros, prazos, resultados especificos]
[Nunca generico. Teste: concorrente poderia usar? Se sim, refaz.]
```

### 3. SUB-HEADLINE

```
[Expande a headline com "como" ou "sem"]
"Como [resultado] em [prazo] sem [objecao principal]"
```

### 4. LEAD / OPENING (escolha um tipo)

**Story Lead:**

```
[Historia pessoal ou do cliente que ilustra a dor]
[Comece in media res - no meio da acao]
[3-5 paragrafos curtos]
```

**Problem Lead:**

```
[Descreva a dor como quem viu acontecer de perto]
[Cena viva, nao descricao generica]
[Use linguagem do publico - VoC]
```

**Secret Lead:**

```
[Revele algo que o mercado nao sabe]
[Posicione como descoberta]
[Crie curiosidade que so se resolve lendo]
```

### 5. PROBLEM AMPLIFICATION

```
[Aprofunde a dor - 3 camadas: superficial → emocional → existencial]
[Mostre que voce ENTENDE (nao que voce julga)]
[Custo de NAO resolver (fantasma emocional)]
"Se voce nao [acao] agora, daqui a [prazo] voce vai [consequencia]..."
```

### 6. MECHANISM REVEAL

```
[Apresente o MECANISMO UNICO - por que funciona diferente]
[Nome proprietario do metodo/sistema]
[Explique visualmente (pode ser desenhado em 30s)]
[Por que tudo que tentaram antes falhou (nao era culpa deles)]
```

### 7. SOLUTION PRESENTATION

```
[O que e o produto/servico]
[Como funciona (3-5 passos simples)]
[Resultados especificos (numeros, prazos, metricas)]
```

### 8. PROOF & CREDIBILITY

```
[Resultados do criador]
[Testimonials de clientes (com numeros)]
[Dados, estudos, autoridade]
[Screenshots, prints, evidencias]
```

### 9. OFFER STACK

```
Item Principal: [Nome] ........... Valor: R$ X.XXX
  - [Beneficio 1]
  - [Beneficio 2]
  - [Beneficio 3]

Bonus #1: [Nome] ................. Valor: R$ XXX
  [Descricao em 1 linha]

Bonus #2: [Nome] ................. Valor: R$ XXX
  [Descricao em 1 linha]

Bonus #3: [Nome] ................. Valor: R$ XXX
  [Descricao em 1 linha]

Valor Total: R$ X.XXX
Voce paga apenas: R$ XXX
```

### 10. GUARANTEE / RISK REVERSAL

```
[Garantia clara e especifica]
[Periodo (7, 15, 30 dias)]
[Sem perguntas / sem burocracia]
[Inverta o risco: "O risco e TODO meu"]
```

### 11. URGENCY / SCARCITY

```
[Real, nao artificial]
[Motivo logico para o prazo]
[Consequencia de esperar]
[Contador se aplicavel]
```

### 12. CTA (Call To Action)

```
[Botao: Verbo + Resultado]
"Quero [resultado] agora"
[Repetir a promessa principal]
[Reforcar garantia]
```

### 13. PS (OBRIGATORIO - Halbert Rule)

```
P.S. [Reforco da promessa principal ou urgencia]
P.P.S. [Bonus extra ou lembrete da garantia]
```

---

## Validation Checklist (Pre-Torriani)

- [ ] Headline tem numero + prazo + resultado?
- [ ] Lead abre com tensao (nao introducao educada)?
- [ ] Mecanismo unico nomeado e visual?
- [ ] Dor descrita em cena viva?
- [ ] Offer stack com valor empilhado?
- [ ] Garantia presente e clara?
- [ ] CTA especifico e binario?
- [ ] PS presente?
- [ ] Zero cliches de coach?
- [ ] Teste do concorrente: copy e proprietaria?
